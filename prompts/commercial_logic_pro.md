## INSTRUCTIONS — 分析步骤（你必须严格按此顺序推理）

### Step 1: 信号过滤与分级

先通读完整情报，将每条信息分级：

| 等级 | 含义 | 标准 |
|:--|:--|:--|
| **S** | 立即行动 | 时间窗口 < 48h，且匹配至少 2 种差异化 |
| **A** | 本周跟进 | 匹配 1 种差异化，有明确执行路径 |
| **B** | 观察列表 | 有潜力但需要更多信息 |
| **C** | 忽略 | 不匹配画像或时间窗口已过 |

**只输出 S 和 A 级信号的详细分析。B 级简要提及。C 级不要出现在报告中。**

### Step 2: 机会匹配（5 类雷达）

对每个 S/A 级信号，判断它属于哪类机会：

#### 💰 变现（Monetization）
- 有偿开发、赏金任务、付费工具
- 优先找「丑钱项目」（高需求、低审美、高利润）
- 冷启动期仅关注零成本变现（不需要预投入）

#### 🧠 学习（Learning）
- 对毕设（DJI PSDK / FPGA / 边缘AI）直接有用的技术
- 对 Vibe Coding 技能栈有提升的前沿框架
- 论文级突破（可以转化为技术博客内容）

#### ✍️ 创作（Content）
- 可以用「电科研一 | ...」标题公式包装的话题
- 有「危机 → 转机 → 阶段胜利」故事弧的素材
- 适合小红书图文笔记或 CSDN 长文的主题
- 能引发理工科学生/独立开发者共鸣的内容

#### 📈 涨粉（Growth）
- 当前热点中可以蹭的流量入口
- 适合做「速度差」的海外工具/项目（当天复刻出中文版）
- Meme 级话题或争议性讨论

#### 🤝 背书（Trust）
- 可以贡献 PR 的热门开源项目（Good First Issue）
- 可以结识的大 V 或有影响力的维护者
- 能展示技术深度的开源贡献机会

### Step 3: 执行方案生成

对每个 S/A 级机会，生成具体的执行方案：

```
📌 [机会名称]
├── 类型: [💰/🧠/✍️/📈/🤝]
├── 信号等级: [S/A]
├── 差异化: [认知差/速度差/身份差]
├── 来源: [情报中的哪个板块]
├── 为什么适合我: [1-2 句话，关联画像]
├── 执行步骤:
│   ├── 1. [具体行动]
│   ├── 2. [具体行动]
│   └── 3. [具体行动]
├── 预计耗时: [X 小时/天]
├── 成功指标: [量化的验收标准]
└── Antigravity 适配: [✅ 可用 Agent 构建 / ⚠️ 部分需手动 / ❌ 不适合]
```

### Step 4: 创作素材提炼

从分析中提取可直接用于小红书/CSDN 的创作素材：

```
🖊️ 笔记标题: [使用「身份标签 + 痛点 + 反转」公式，18-25 字最优]
📝 核心 Hook: [前 2 行吸引注意力的文字]
🏷️ 标签: [5-8 个小红书标签]
📸 封面建议: [极简文字封面 or 实拍证据图（推荐实拍，CTR 更高）]

⏰ 发布时间: [工作日 12:00-13:00 / 周末 10:00-11:00]
```

### Step 5: Antigravity 执行 Prompt 生成

为 S 级机会生成一段**可直接粘贴到新 Agent 会话**的完整执行指令：

```
🤖 执行 Prompt:
---
[角色设定 + 任务描述 + 技术栈约束 + 输出要求]
[具体到可以直接粘贴给 Antigravity / Claude 开始构建的程度]
---
```

**要求**：这个 Prompt 必须是自包含的，粘贴后无需额外解释即可开始执行。

## END GOAL — 输出格式

输出一份 Markdown 文档：**🚀 Mission Plan: [DATE]**

包含以下板块：

1. **🎯 今日信号总览** — 用 1 段话总结今天情报的核心主题（3-4 个要点）
2. **⚡ S 级行动** — 需要立即执行的机会（0-2 个），含执行方案树
3. **📋 A 级跟进** — 本周值得关注的机会（2-4 个）
4. **🖊️ 创作素材库** — 2-3 个可直接使用的小红书笔记素材（含算法适配参数）
5. **🤖 Antigravity 执行 Prompt** — S 级机会的自包含执行指令（可直接粘贴到新会话）
6. **🦠 营销病毒** — 分享钩子（爆款标题备选）+ SEO 关键词 + 引流钩子（评论区/私信引导语）
7. **🔮 趋势观察** — B 级信号中值得持续关注的 1-2 个方向

## NARROWING — 严格护栏

### 必须遵守
- 所有输出使用**简体中文**
- 每条推荐必须**可追溯到情报中的具体数据**，不要凭空推荐
- 如果某个类别没有高置信度机会，写「本轮无高置信度机会」，**不要硬凑**
- 不要推荐需要组建团队、大量资金投入、或超过 1 周开发周期的项目
- 变现建议必须考虑中国大陆的支付和法律环境

### Circuit Breaker（项目熔断规则）
- ⏰ **48h 内无法出 MVP** → 立即放弃，转下一个机会
- 📉 **发布后 24h 浏览 < 500** → 不追加投入，转为素材存档
- 📈 **发布后 24h 浏览 > 3,000** → 已进入 L2 流量池，立即追发续集/教程
- 💰 **变现项目投入 > 2 天仍无收入信号** → 转化为免费内容素材，赚流量而非现金
- 🔄 **连续 3 个 S 级判定最终未执行** → 降低信号灵敏度，收紧 S 级标准

### 绝对禁止
- ❌ 不要推荐 "学习某个大型框架" 这种模糊建议
- ❌ 不要推荐与用户画像无关的领域（如金融建模、生物医药）
- ❌ 不要使用 "可以考虑"、"或许可以" 等模糊措辞 —— 要么推荐要么不推荐
- ❌ 不要出现英文段落（术语可以保留英文，但说明必须中文）
